# Rockfall Prediction Model

This project is a Flask web app for predicting rockfall risk using a trained machine learning model. It includes dataset exploration, model training, and a simple prediction UI.

## Dataset

- File: `Rockfall_Risk_Data.csv`
- The dataset contains geological and environmental features for rockfall risk prediction.
- Example feature columns:
  - `slope_angle`
  - `rock_strength`
  - `rainfall_mm`
  - `seismic_activity`
  - `temperature_variation`
  - `soil_moisture`
  - `vegetation_cover`
  - `mining_depth`
  - `blasting_frequency`
  - `weathering_index`
  - `human_activity_index`
  - `rainfall_intensity`
  - `slope_stability_index`
- Target column: `rockfall_risk` with values like `Low`, `Medium`, and `High`.

## Model

- File: `model.pkl`
- The Flask app loads the serialized model and uses it to predict a risk category from 13 feature inputs.
- The model is likely trained in `model_training.py` and saved as `model.pkl`.

## Web App

- Entry point: `app.py`
- Templates: `templates/index.html`
- Static assets: `static/` (plot image generated by the app)

### How it works

- The home page renders a risk distribution plot and shows the prediction form.
- When the form is submitted, the app sends the feature values to `/predict`.
- The model returns a risk label, which is displayed in the UI.

## Results

- The app displays a bar chart of the rockfall risk distribution.
- It returns one of the risk categories: `Low Risk`, `Medium Risk`, or `High Risk`.
- The prediction result is shown directly on the form page.

## Run locally

1. Activate the virtual environment:
   ```powershell
   .\.venv\Scripts\Activate.ps1
   ```
2. Install dependencies if needed:
   ```powershell
   python -m pip install -r requirements.txt
   ```
3. Start the app:
   ```powershell
   python app.py
   ```
4. Open the browser:
   ```text
   http://127.0.0.1:5000
   ```

## Notes

- If you do not have a `requirements.txt`, generate one with:
  ```powershell
  python -m pip freeze > requirements.txt
  ```
- Add a GitHub remote and push after initialization.
